SL-SRE Onboarding Portal

Everything you need to get up and running on the Service Lifecycle SRE team. Work through each section in order.

Access Pillar Toolchain Channels Checklist

1 — Getting Access

Raise access requests in order — some depend on others. Managed via app-interface MRs unless otherwise noted.

1
Red Hat SSO & GitLab.cee
Your Red Hat corporate account is your SSO. Use it to log into gitlab.cee.redhat.com. You'll need a GitLab account before any app-interface MRs can be raised. Request group membership for hcm-service-lifecycle from your manager.
GitLab.cee → hcm-service-lifecycle group
2
app-interface — SL-SRE User File
Your onboarding manager will raise an app-interface MR to create your user file under data/teams/sl-sre/. This file grants access to Jira, PagerDuty, Unleash roles, and LDAP groups. Review the PR when raised — your name, email, and timezone must be correct. Branch pattern: teams/SLSRE-XXX-your-name-onboarding.
app-interface → Developer Guidelines
3
Jira — SLSRE Project
Sprint planning, epic tracking, and on-call incident management all live in Jira. Access is granted via app-interface user file. Once provisioned, bookmark the board and the current sprint.
Sprint Board → SLSRE Project
4
PagerDuty — On-Call Schedules
SRE on-call is managed via PagerDuty. Your app-interface user file will add you to the relevant escalation policies. Install the PagerDuty mobile app and configure your notification preferences. Talk to your manager about which on-call rota you will join and when you go live.
PagerDuty → On-Call Schedules
5
OCM Backplane — Cluster Access
Cluster access is via ocm backplane. You need the OCM CLI and backplane plugin installed. Elevation uses ocm backplane elevate "reason" -- get pods -n namespace — note: do NOT include kubectl after --, backplane wraps it internally.
ocm login --token="$(cat ~/.ocm-token)" ocm backplane login <cluster-id> ocm backplane elevate "investigating alert" -- get pods -n uhc-production
Backplane Docs → OCM SOPs
6
Konflux — CI/CD Platform Access
Konflux is Red Hat's internal CI/CD platform used for building and releasing services. Follow the official getting-access guide to request membership in the mos-lpsre-tenant Konflux tenant.
Getting Access → Konflux Docs
7
RHOBS — Observability Platform
Metrics and alerting are migrating to RHOBS (hcp tenant). Access is via the rhobs-ocm-ingestion-prod credential secret in app-sre-observability-per-cluster. Read the handbook before touching RHOBS config.
RHOBS Handbook → SLSRE-221 (RHOBS.next)

4 — SRE Toolchain Setup

Install and configure the SRE toolchain in this order. All tools are required for full team productivity.

🔐
OCM CLI + Backplane
Core cluster access tool. Required for all production investigations. Remember the backplane elevate syntax — no kubectl after --.
brew install openshift-clients ocm login --token="..." ocm backplane login <cluster-id>
Backplane Docs
🤖
Claude Code + rosa-claude-plugins
AI-augmented SRE workflows. Claude Code with the rosa-claude-plugins package gives you context-aware skills for OCM CS, ACM observability, OADP, and SLSRE-232 onboarding.
npm install -g @anthropic-ai/claude-code cd ~/.claude git clone git@github.com:openshift-service-lifecycle-engineering/rosa-claude-plugins.git
rosa-claude-plugins →
🔍
web-rca-cli (wrca)
CLI tool for Red Hat's web-rca incident management system. Token resolution uses ocm token which returns a live bearer JWT directly. Built as a Python Click package with Rich formatting.
pip install wrca # or clone from internal GitLab ocm token # get live bearer token wrca incidents # list recent incidents
web-rca GitLab → RCA Jira Issues
📚
ask-sre (RAG pipeline)
Semantic search over ops-sops, ROSA KCS, and openshift-docs corpora. Requires Poetry + Python 3.12 (3.14 incompatible). Registered as an MCP server in Claude Code.
curl -sSL https://install.python-poetry.org | python3.12 - git clone <ask-sre-repo> poetry install && poetry run ask-sre index
ask-sre repo →
⚡
Konflux CLI + Access
Used for OLM→PKO migration (SLSRE-581), ADO delivery, and service releases. Tenant: mos-lpsre-tenant. Request access via the getting-access guide first.
Getting Access → Konflux Docs
🌐
Cora + Ambient AI Platform
Cora is the Red Hat internal AI assistant. Ambient is the SRE automation platform. Both are part of the five-tool AI ecosystem: Cora → Claude Code → ask-sre → Managed Notifications MCP → SRE Agent.
Cora → Ambient Jira Issues
